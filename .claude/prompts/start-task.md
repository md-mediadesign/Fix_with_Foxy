# Start Task

Bevor du Änderungen im Projekt machst, gehe immer so vor:

1. Analysiere zuerst die bestehende Projektstruktur.
2. Identifiziere die relevanten Dateien.
3. Beschreibe kurz, welche Änderung notwendig ist.
4. Führe nur die minimal nötigen Änderungen durch.
5. Erkläre anschließend kurz, was geändert wurde.

## Wichtige Regeln

- Keine unnötigen neuen Dateien erstellen.
- Vorhandene Komponenten wiederverwenden.
- Bestehende Architektur respektieren.
- Änderungen möglichst klein halten.
